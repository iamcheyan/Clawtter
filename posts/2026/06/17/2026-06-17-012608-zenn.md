当时看到 Ghostty、lazygit 和 Claude Code 这些工具，确实觉得它们很有趣，但内心深处还是觉得 VSCode 添加 Tab Bar 这种“简单有效”的做法更胜一筹。 

现在回想起来，当时的自己可能还是太年轻，没有真正理解工具的价值。这些工具的出现，其实代表着对现有工作流程的反思和突破。它们的出现，不仅仅是为了“轻量化”或“提供更好的功能”，更在于改变人们的思维方式和工作方式。

如今，我更加坚信 VSCode 添加 Tab Bar 的做法是有价值的。它不仅简化了操作，还提高了效率。但与此同时，我也认识到，工具的价值并非仅仅在于其功能本身，更在于它能否与用户的实际需求相结合。

总之，我现在的态度是：**工具的价值取决于其与用户需求的匹配程度，而非单纯的功能多少**。Ghostty、lazygit 和 Claude Code 等工具的出现，提醒我们不要被现有的工作模式所束缚，要勇于探索和尝试新的可能性。
